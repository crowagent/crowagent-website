## 2 · Design System (tokens, colour, spacing, motion)

### 2.1 Tokens

- `styles.css` defines **340 CSS custom properties** in `:root` — solid foundation, consistent naming (`--obsidian`, `--navy`, `--teal`, `--teal-d`, `--lime`, `--sky`, `--steel`, `--cloud`, `--space-*`, `--dur-*`, `--ease-*`).
- All required Material 3 Expressive motion tokens present (`--motion-spatial-hero`, `--dur-base`, `--ease-signature`, etc.).

### 2.2 Hardcoded hex outside `:root`

- `styles.css`: 78 raw hex literals. Most are inside the `:root` token definitions block (legitimate — that *is* where hex must live). A finer-grained sweep is recommended for Session B Phase 0 (grep for `#[0-9a-fA-F]{3,8}` outside lines 1–500 token block).
- `index.html` + `scripts.js`: **6 hardcoded hex literals** outside `:root`. These violate WP §A4 "zero hardcoded hex outside token block" and must be tokenised in Session B.
- `Assets/css/ca-hero-finishing.css`: 1 raw rgba `rgba(184, 204, 224, 0.4)` at line 59 — should be tokenised as `--carousel-dot-idle` or similar.

### 2.3 CTA pattern

- Spec: `background: var(--teal)` + `color: var(--obsidian)`.
- Sampling of `.sv-btn--primary`, `.cta-primary`, `.btn-primary` rules in `styles.css`: pattern is followed in the primary set. `Assets/css/ca-hero-finishing.css` and the carousel dot rules use `var(--teal)` with default focus outline of `var(--teal)` — consistent.
- One visible legacy class `.crow-carousel-dot[aria-selected="true"] { background: var(--teal) }` — correct.
- One outstanding concern from prior session: "white-on-teal" regression noted in `feedback_no_agent_trust_pixel_verify.md`. This needs Session B Phase 0 pixel-verification on the live `localhost:8092` build, not a desk-check.

### 2.4 Motion

- `prefers-reduced-motion: reduce` is respected in `js/modules/ca-rotator.js` (line 18, 28). Spot-checks on other modules indicate coverage; full audit deferred to Session B Phase 0.
- `animation-timeline: view()` usage and `@supports` fallback — partial coverage; some sections still rely on IntersectionObserver only. Acceptable per spec but should be unified in Session B.
