# DESIGN SYSTEM COMPONENT REGISTRY
**Decision-of-record:** 2026-05-21

## Canonical primitives — `.sv-*` namespace

All new HTML MUST use these primitives. Any other class is deprecated.

### Layout
- `.sv-container` — base, max-width 1200px, inline-margin auto, padding-inline clamp
- `.sv-container--standard` — 880px (prose-friendly)
- `.sv-container--wide` — 1320px (hero-wide)
- `.sv-container--text` — 720px (article body)
- `.sv-section` — vertical-rhythm section wrapper
- `.sv-stack` — vertical flow primitive (`.sv-stack--gap-N` for spacing)
- `.sv-cluster` — horizontal flow with wrap
- `.sv-grid` — auto-fit grid (`.sv-grid--sm/md/lg/xl` for column hints)

### Surfaces
- `.sv-card` — base card; `.sv-card--accent` for emphasised border
- `.sv-card__eyebrow` / `.sv-card__title` / `.sv-card__body` — slots

### Interactive
- `.sv-btn` — base button
- `.sv-btn--primary` — solid teal
- `.sv-btn--secondary` — outlined
- `.sv-btn--ghost` — text-only
- `.sv-btn--sm` / `.sv-btn--md` / `.sv-btn--lg` — size modifiers
- `.sv-btn--icon` — square icon button

### Type
- `.sv-eyebrow` — section eyebrow (uppercase teal small-caps)
- `.sv-h1` / `.sv-h2` / `.sv-h3` — explicit heading classes (rarely needed; semantic h1/h2/h3 carry style)

## Lint gate

Sovereign-sheriff enforces the following at every CSS commit:
- No new `var(----foo)` (broken token pattern)
- No new hardcoded `font-size: Npx`, `margin: Npx`, `padding: Npx`, `gap: Npx` literals in author CSS (Assets/css/sovereign-primitives.css + crowagent-brand-tokens.css are exempt as token sources)
- No new raw 6-digit hex literals in author CSS
- No new `z-index: N` literals (use `var(--z-*)`)

## Palette policy

**Monochrome teal (Stripe pattern), committed 2026-05-21.**

All product accents (CrowCyber, CrowCash, CrowMark, CrowESG, CrowAgent Core, CSRD Checker) use `var(--teal)` for accents. Products are distinguished by icon + name + content, NOT by per-product accent colour. Reasoning:
- Aligns with TRUE-CANONICAL-LOGO.svg ocean/teal gradient
- Pre-launch charter: "no fake customer accents"
- Stripe pattern: monochrome reduces visual noise, increases brand cohesion

To restore per-product palette in the future: delete the `Wave 12 PALETTE DECISION` block at the end of styles.css.

## Hero archetype

Single canonical pattern (defined inline in styles.css until a dedicated `.sv-hero` primitive is built):
- Top: announce bar (1 line, full-width)
- Below: nav (injected by nav-inject.js)
- Body:
  - Eyebrow chip (outlined pill, teal small-caps)
  - H1 (display font, large clamp)
  - Subtitle (body font, max 56ch)
  - CTA row (2 buttons max: primary + secondary)
  - Optional trust microcopy below CTAs

Intel/tracker pages and glossary pages now share this archetype (UI-14 fix). Per-product hero backgrounds: monochrome dark gradient only (UI-05 fix).
